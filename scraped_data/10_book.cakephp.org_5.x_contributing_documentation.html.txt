Contributing to the documentation is simple. The files are hosted on https://github.com/cakephp/docs. Feel free to fork the repo, add your changes/improvements/translations and give back by issuing a pull request. You can even edit the docs online with GitHub, without ever downloading the files -- the "Edit this page" link on any given page will direct you to GitHub's online editor for that page.

The CakePHP documentation is built with VitePress, a static site generator powered by Vue and Vite. The documentation is continuously integrated and deployed after each pull request is merged.

Email the docs team (docs at cakephp dot org) or hop on IRC (#cakephp on freenode) to discuss any translation efforts you would like to participate in.

We want to provide translations that are as complete as possible. However, there may be times where a translation file is not up-to-date. You should always consider the English version as the authoritative version.

If your language is not in the current languages, please contact us through Github and we will consider creating a skeleton folder for it. The following sections are the first one you should consider translating as these files don't change often:

The structure of all language folders should mirror the English folder structure. If the structure changes for the English version, we should apply those changes in the other languages.

For example, if a new English file is created in en/file.md, we should:

Add the file in all other languages : fr/file.md, zh/file.md, ...

Delete the content, but keep the frontmatter and add a translation notice. The following note will be added while nobody has translated the file:

Adding Technical Terms

If cspell flags legitimate technical terms (class names, CakePHP-specific terms), add them to the words array in .github/cspell.json.

We use markdownlint via markdownlint-cli2 to maintain consistent markdown formatting:

Auto-fix Limitations

The --fix flag can automatically correct many formatting issues, but not all. Review changes before committing.

External Links

The link checker only validates internal markdown links. External URLs (http://, https://) are not checked.

Known Issues Baseline

The repository maintains a baseline of known broken links in .github/linkchecker-baseline.json. When you run the checker with --baseline, it filters out these known issues and only reports new problems. This is useful when working on specific changes without being overwhelmed by pre-existing issues.
